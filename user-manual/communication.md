# GARY-COMMUNICATION(1) — User Manual

## NAME

**gary-communication** — how to reach me and preferred formats

## SYNOPSIS

```
gary communication [--channels] [--written]
```

## OPTIONS

**--channels**
: Preferred communication channels and expected turnaround:


| Channel | Turnaround   | When to Use                                                                                                |
| ------- | ------------ | ---------------------------------------------------------------------------------------------------------- |
| Text    | Fastest      | Need a response now                                                                                        |
| Slack   | 1–8 hours    | Standard async; OK to send/receive evenings and weekends — reply next business day                         |
| Email   | 8–24 hours   | Lower priority or longer-form topics                                                                       |
| Phone   | ECONNREFUSED | tldr: text first — routed to voicemail unless you're in my contacts; voicemail usually goes to `/dev/null` |


**--written**
: There are parts, but certainly not all, of the Amazon culture that are
  valuable, and one of these is using a written "one pager" in
  communications. The act of writing forces clarity, it can be more easily
  disseminated in a remote team, can be archived, and is typically faster to
  process in a meeting setting. In AI-forward organizations it can be easily
  consumed and incorporated by an LLM.
